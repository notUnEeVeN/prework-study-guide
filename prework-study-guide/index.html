HTML

  • IGNORE ALL PERIODS IN THE TAGS, THEY WERE ADDED TO DISABLE THEM SO THEY CAN APPEAR IN THE TEXT
  • The head element contains information about the webpage used for the backend/broswer/SEO, and is not seen by the user
  • The body element represents the content that will be visible to the user
  • HTML will be the foundational structure of your website, the first step
  • Many HTML commands are semantic, allowing other coders to easily understand the function of a certain command
  • HTML commands are enclosed in <>, and often will begin with an start and end tag, such as <.p> and <./p> (without the periods) which would create a paragraph
  • DOCTYPE html is a decleration that helps web browsers display web pages correctly
  • Common HTML tags: <.h1> through <.h6> for headers, <.p> for a paragraph, <.a> for a hyperlink, <.img> to embed images, <.div> is a container unit that encapsulates other page elements and divides the HTML document into sections, <.ul> and <.li>: to create unordered lists

Git

  • Git is a version control system that tracks the changes and version history of a source code during software development
  • Git allows multiple developers to work on the same code concurrently
  • Some common bit commands you will use are: git clone <.repo> creates a copy of a remote repository on your local machine
  • git checkout -b branch-name: creates a new branch and switches to it
  • git status: shows the status of changes as untracked, modified, or stages,
  • gid add <.file>: adds an untracked or modified file to the staging area
  • give commit -m <.message>: commits the staged changes with a descriptive message
  • git push: pushes the commited changes to the remote repository
  • git pull: fetches changes from the remote repository and merges them into the current branch
  • git branch: lists all the local branches
  • You will use git by creating your own 'branch' isolated from the main code to add/edit code, without it impacting the rest of the main code, when you have completed your work, it gets checked by other developers on the team, and then once approved, merged into the main code, and your branch is deleted.
